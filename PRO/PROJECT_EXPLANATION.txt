CARDIOSENSE PROJECT EXPLANATION (FINAL DELIVERABLE)

1) PROJECT OVERVIEW
CardioSense is a heart disease risk analysis and prediction system. We take a real-world heart disease dataset, apply statistical analysis and probability modeling, and then build a logistic regression model for prediction. We also provide a Streamlit web app that shows results, charts, and a simple risk estimator for new inputs.

2) DATA SOURCE
Dataset: Heart Disease Dataset (Kaggle)
File: heart.csv
Rows: 1025
Columns: 14
Target: target (1 = heart disease present, 0 = no disease)

3) DATASET FEATURE NAMES (FULL NAMES + MEANING)
Numeric features (scaled in the model):
- age: Age in years.
- trestbps: Resting blood pressure (mm Hg).
- chol: Serum cholesterol (mg/dl).
- thalach: Maximum heart rate achieved.
- oldpeak: ST depression induced by exercise relative to rest.

Categorical features (one-hot encoded in the model):
- sex: Sex (1 = male, 0 = female).
- cp: Chest pain type (coded 0-3; typical UCI mapping: 0=typical angina, 1=atypical angina, 2=non-anginal pain, 3=asymptomatic).
- fbs: Fasting blood sugar > 120 mg/dl (1 = true, 0 = false).
- restecg: Resting ECG results (coded 0-2; typical mapping: 0=normal, 1=ST-T wave abnormality, 2=left ventricular hypertrophy).
- exang: Exercise induced angina (1 = yes, 0 = no).
- slope: Slope of peak exercise ST segment (coded 0-2; typical mapping: 0=upsloping, 1=flat, 2=downsloping).
- ca: Number of major vessels colored by fluoroscopy (coded 0-4).
- thal: Thalassemia / thallium stress test result (coded 0-3; typical mapping: 0=unknown, 1=normal, 2=fixed defect, 3=reversible defect).

Target:
- target: Heart disease indicator (1 = present, 0 = absent).

4) FEATURES IMPLEMENTED (WHAT WE BUILT)
A) Core analysis pipeline
- Data loading and dataset summary (rows, columns, missing values, target rate).
- Descriptive statistics table (count, mean, std, min, quartiles, max).
- 95% confidence intervals for numeric columns using a t-distribution.
- Probability calculations:
  * P(target=1) overall.
  * P(target=1 | sex) for male and female.
  * Binomial example: P(X=7) for n=10, p=target_rate.
  * Normal model for cholesterol: P(chol > 240).

B) Visual analysis
- Target count bar chart.
- Histograms with KDE for age, chol, trestbps, thalach, oldpeak.
- Correlation heatmap for numeric features + target.
- Boxplot: cholesterol by target.

C) Predictive modeling
- Train/test split (80/20), stratified by target.
- Preprocessing:
  * StandardScaler for numeric features.
  * OneHotEncoder for categorical features.
- Logistic regression model (max_iter=1000).
- Evaluation metrics:
  * Accuracy, Precision, Recall, F1, ROC AUC.
  * Confusion matrix.
  * ROC curve points for plotting.

D) Streamlit web application
- Tabs: Overview, Descriptive Stats, Probability, Visuals, Model.
- Clean UI with custom styling and summary cards.
- Metrics display with confusion matrix and ROC curve.
- Risk prediction form (sliders + dropdowns) to estimate probability for new inputs.
- Cached data, plots, and model for faster loading.

5) FILES AND ROLES (FINAL DELIVERABLES)

- PRO/code_submission/src/app.py: Streamlit app and UI logic.

1) pip install -r requirements.txt

3) streamlit run src/app.py
4) python report/build_report.py (optional report build)
